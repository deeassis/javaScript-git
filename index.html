Seja Bem Vindo ao desenvolvimento de página da web Aula 02, nessa aula aprendemos mais sobre o GitHub e as formas de comitar os códigos!

GitHub é uma plataforma de hospedagem de código-fonte e colaboração baseada em Git. Permite que desenvolvedores e equipes armazenem, gerenciem e compartilhem projetos de software de maneira eficiente. Além disso, facilita o controle de versões, colaboração entre desenvolvedores e integração contínua, sendo amplamente utilizada na comunidade de desenvolvimento de software para controle de código-fonte e gerenciamento de projetos.

GitHub: Comandos
  • git status
    • O comando git status é utilizado para verificar o estado atual do seu repositório Git. Ele mostra informações como arquivos modificados, arquivos adicionados, arquivos removidos, entre outros. É útil para entender o que foi alterado desde o último commit.
  • git add .
    • O comando git add . adiciona todas as alterações no seu diretório de trabalho (working directory) ao índice (staging area) do Git. Isso prepara os arquivos modificados para serem incluídos no próximo commit.
  • git status
    • Após o git add ., executar git status novamente mostra os arquivos que foram adicionados à staging area. Esses arquivos agora estão prontos para serem commitados.
  • git commit -m "adicionando imagem"
    • O comando git commit -m "adicionando imagem" realiza o commit das alterações adicionadas à staging area. A opção -m permite adicionar uma mensagem de commit inline, que descreve sucintamente as mudanças feitas no commit. No exemplo dado, a mensagem é "adicionando imagem".
  • git push origin main
    • O comando git push origin main envia (push) todos os commits locais da branch main para o repositório remoto chamado origin. Isso faz com que as alterações sejam atualizadas no repositório compartilhado, permitindo que outros membros da equipe ou colaboradores vejam e acessem as últimas mudanças.

Esses comandos são parte fundamental no fluxo de trabalho com Git, desde o acompanhamento de mudanças até o compartilhamento delas com outros colaboradores no repositório remoto.

GitHub: Comandos - Main
  • git status
    • O comando git status é utilizado para verificar o estado atual do seu repositório Git. Ele mostra informações como arquivos modificados, arquivos adicionados, arquivos removidos e a branch atual.
  • git checkout main
    • O comando git checkout main muda o ramo atual de trabalho para a branch main. O checkout também pode ser usado para criar novas branches ou alternar entre diferentes branches existentes.
  • git pull origin main front-2
    • O comando git pull origin main front-2 realiza duas operações: primeiro, ele faz um fetch dos commits remotos da branch main e front-2 do repositório remoto origin; em seguida, realiza um merge desses commits na sua branch local atual (provavelmente main). Isso atualiza sua branch local com as alterações mais recentes das branches main e front-2 no repositório remoto.
  • git push origin main
    • O comando git push origin main envia (push) todos os commits locais da branch main para a branch main do repositório remoto chamado origin. Isso atualiza o repositório remoto com as suas alterações locais na branch main.

Estes comandos são típicos em um fluxo de trabalho colaborativo com Git, onde você verifica o estado, muda entre branches, sincroniza com o repositório remoto e envia suas alterações para o servidor compartilhado. Certifique-se de entender o contexto do seu projeto e como esses comandos afetam o seu ambiente de desenvolvimento e colaboração.
